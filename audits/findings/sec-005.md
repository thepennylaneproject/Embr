# Finding: sec-005

> **Status:** open | **Severity:** High | **Priority:** P1 | **Type:** security | **Confidence:** Evidence

## Title

IDOR in wallet endpoints—no ownership verification

## Description

The WalletController.getWallet() method retrieves wallet data but does not verify that the requesting user actually owns the wallet. If route parameters accept wallet IDs or user IDs, an attacker could enumerate wallets and access other users' transaction histories, balances, and financial data.

## Reproduction Steps

1. Authenticate as User A (obtain JWT)
2. Call endpoint: `GET /api/wallet`
3. Inspect response, note wallet ID (if exposed)
4. Attempt to call `GET /api/wallet/:walletId` with another wallet ID
5. If wallet belongs to User B, you receive their transaction history and balance
6. Repeat to enumerate all wallets

## Impact

**Financial Data Exposure:**
- User wallet balances (who has money)
- Transaction history (payment patterns, frequency)
- Recipient information (who they pay)
- Account types and payment methods
- Tax/financial records

**Attack Scenario:**
An attacker can:
1. Enumerate all wallet IDs (sequential or from other responses)
2. Identify high-balance accounts
3. Monitor transaction patterns
4. Track payment flows between users
5. Identify creators by transaction volume
6. Perform financial analysis on the platform

**Fraud Potential:**
- Identify users worth targeting for account takeover
- Monitor victim's financial activity
- Track money laundering patterns
- Correlate users across accounts

**Compliance Impact:**
- GDPR: Unauthorized access to personal data
- PCI DSS: Wallet data exposure
- Financial regulations: Transaction data breach

## Suggested Fix

**Approach:** Add ownership verification before returning wallet data

**Option 1 (Recommended):** Custom decorator/middleware
```typescript
@CheckOwnership('walletId', 'userId')
async getWallet(@GetUser('id') userId: string) {
  return this.walletService.getWallet(userId);
}
```

**Option 2:** Service-level validation
```typescript
async getWallet(walletId: string, userId: string) {
  const wallet = await this.prisma.wallet.findUnique({
    where: { id: walletId }
  });
  
  if (wallet.userId !== userId) {
    throw new ForbiddenException('Access denied');
  }
  
  return wallet;
}
```

**Affected files:**
- `apps/api/src/core/monetization/controllers/wallet.controller.ts`
- `apps/api/src/core/monetization/services/wallet.service.ts`
- Create decorator: `@CheckOwnership()` (reusable pattern)

**Effort:** 1-2 hours (add validation, create decorator)

**Risk:** Low — defensive fix, no functional changes

## Enhancement Notes

**IDOR Prevention Pattern:**

```typescript
// Reusable decorator
@Injectable()
export class OwnershipGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    const request = context.switchToHttp().getRequest();
    const { walletId } = request.params;
    const { id: userId } = request.user;
    
    // Check wallet owner
    return request.walletService.verifyOwnership(walletId, userId);
  }
}

// Usage
@Get(':walletId')
@UseGuards(OwnershipGuard)
async getWallet(@Param('walletId') walletId: string) {
  return this.walletService.getWallet(walletId);
}
```

**Endpoints to Audit for IDOR:**

- [ ] GET /api/wallet (any user's wallet)
- [ ] GET /api/wallet/:walletId (specific wallet)
- [ ] GET /api/wallet/transactions (any user's transactions)
- [ ] GET /api/gigs/:gigId (gig details - only creator/applicants)
- [ ] GET /api/messages/:conversationId (conversation - only participants)
- [ ] GET /api/profile/:userId (user profile - public vs private data)
- [ ] PATCH /api/user/:userId (settings - only self)
- [ ] DELETE /api/account/:userId (account deletion - only self)

**Common IDOR Patterns to Check:**

1. **Resource ID in URL:** `/api/resource/:id` → Verify ownership
2. **User ID parameter:** `/api/users/:userId/data` → Verify equals current user
3. **Foreign key access:** `/api/user/:userId/transactions` → Verify ownership
4. **Enumeration attacks:** Sequential IDs (1, 2, 3, ...) can be brute-forced
