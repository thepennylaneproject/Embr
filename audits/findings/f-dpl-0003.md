## Title

CI declares checks that are not reliably runnable in current repo state

## Description

CI runs API lint and test steps, but API package scripts rely on tooling not declared in apps/api/package.json (e.g., jest) and no API ESLint config is present while only web has an ESLint config file. This creates a high likelihood of flaky or permanently failing CI, reducing trust in pipeline status.
