## Title

No centralized production logging or external error reporting path

## Description

Logging remains mostly console/Nest logger output with no integrated sink (Sentry/OpenTelemetry/structured log pipeline). Web reliability tracking is localStorage-only and analytics endpoint currently logs events instead of persisting them.

## Impact

Production incidents are difficult to detect and correlate across services; developers remain blind to many failures until user reports.


## Suggested Fix

**Approach:** Adopt structured logging and external error/trace sink, wire web and API error paths to it, and persist analytics/reliability signals server-side.

**Affected files:** `apps/api/src/main.ts` `apps/web/src/components/ErrorBoundary.tsx` `apps/web/src/lib/reliability.ts` `apps/web/src/pages/api/analytics/events.ts`

**Effort:** medium

**Risk:** Telemetry additions can increase noise/cost; define sampling and PII redaction rules up front.
