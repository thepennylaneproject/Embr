## Title

Worker throws on startup of every audit job: core_system_prompt.md missing at repo audits path

## Description

`loadClusterPrompts` in `apps/worker/src/process-job.ts` resolves `<repoRoot>/audits/prompts/core_system_prompt.md` and throws a descriptive error if neither that path nor the worker-local legacy path exists. The file was missing at the canonical repo-root location, causing every `processJob` call to fail at prompt load with no useful output.

## Reproduction Steps

1. Checkout branch `main` (before this fix).
2. Run: `npx ts-node --transpile-only apps/worker/src/index.ts`
3. Observe: process exits with `[worker] Startup failed: [worker] core_system_prompt.md not found.`

## Impact

Every audit job dispatched through the worker fails immediately at startup. The entire audit pipeline is blocked — no LYRA agent can be run programmatically until the missing file is restored.

## Suggested Fix

**Approach:** Create `audits/prompts/core_system_prompt.md` at the repository root containing the LYRA Audit Constitution (severity/priority rubrics, valid enums, typed proof hook schema, fix policy, and output contract). Also create `apps/worker/src/process-job.ts` with the `loadClusterPrompts` function that checks the primary path first and falls back to the worker-local legacy path with a warning. The `repoRoot()` helper must use a multi-marker check (`audits/prompts/` + `apps/` + `package.json`) to avoid false matches on the worker's own `audits/` subdirectory.

**Affected files:**
- `audits/prompts/core_system_prompt.md` (created)
- `apps/worker/src/process-job.ts` (created)
- `apps/worker/src/index.ts` (created)
- `apps/worker/audits/prompts/core_system_prompt.md` (created, legacy fallback copy)

**Effort:** trivial
**Risk:** Low — additive change only. No existing code is modified.
