## Title

CI and deploy web builds omit required NEXT_PUBLIC_WS_URL

## Description

apps/web/next.config.js now throws when NEXT_PUBLIC_WS_URL is missing, but the CI Build Web step only sets NEXT_PUBLIC_API_URL and the deploy Build Web step sets neither public URL. In a clean GitHub Actions environment without apps/web/.env.local, next build will fail before deployment artifacts are produced.

## Impact

Main-branch deploys can be blocked by environment validation even when application code compiles locally with a checked-in or local .env file.

## Suggested fix

Provide NEXT_PUBLIC_WS_URL and NEXT_PUBLIC_API_URL in CI/deploy build environments, preferably via repository/environment variables or explicit workflow env entries.

**Affected files:** `.github/workflows/ci.yml`, `.github/workflows/deploy.yml`, `apps/web/next.config.js`

## Proof hooks

- **[code_ref]** Deploy Build Web runs npm --prefix apps/web run build without NEXT_PUBLIC_* env.
