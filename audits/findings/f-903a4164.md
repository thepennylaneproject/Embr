## Title

CI and deploy web builds omit required NEXT_PUBLIC_WS_URL

## Description

CI and deploy web build jobs now provide both required public web environment variables. CI uses local placeholders for NEXT_PUBLIC_API_URL and NEXT_PUBLIC_WS_URL; deploy reads NEXT_PUBLIC_API_URL and NEXT_PUBLIC_WS_URL from GitHub environment variables, matching the web build-time validation contract.

## Impact

Clean CI/deploy web builds no longer fail because NEXT_PUBLIC_WS_URL is absent from the build environment.

## Suggested fix

Provide NEXT_PUBLIC_WS_URL and NEXT_PUBLIC_API_URL in CI/deploy build environments, preferably via repository/environment variables or explicit workflow env entries.

**Affected files:** `.github/workflows/ci.yml`, `.github/workflows/deploy.yml`, `apps/web/next.config.js`
