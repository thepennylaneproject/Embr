# Finding: inv-002

> **Status:** open | **Severity:** Critical | **Priority:** P0 | **Type:** security | **Confidence:** Evidence

## Title

Critical CVEs in mobile app via unofficial react-native-socket.io-client (CVSS 9.8, 9.8, 9.4)

## Description

`apps/mobile` uses `react-native-socket.io-client` — an unofficial, unmaintained third-party package — instead of the canonical `socket.io-client` package from the Socket.IO team. This unofficial package vendors an outdated copy of `xmlhttprequest-ssl` and `socket.io-parser` which carry three critical CVEs:

| CVE | CVSS | Description |
|-----|------|-------------|
| GHSA-qm95-pgcg-qqfq | 9.8 | socket.io-parser: insufficient validation when decoding packets (RCE-class) |
| GHSA-h4j5-c7cj-74xg | 9.8 | xmlhttprequest-ssl: arbitrary code injection |
| GHSA-72mh-269x-7mh5 | 9.4 | xmlhttprequest-ssl: improper certificate validation |

The official `socket.io-client` package (maintained by the Socket.IO team) does not have these vulnerabilities and is compatible with the NestJS Socket.IO gateway already in use.

## Reproduction Steps

1. `npm audit --workspace apps/mobile` — observe critical findings for `socket.io-parser` and `xmlhttprequest-ssl`.
2. `npm ls react-native-socket.io-client --workspace apps/mobile` — confirm it is the transitive source.

## Impact

**Severity: CRITICAL**

An RCE-class vulnerability (GHSA-qm95-pgcg-qqfq) means a malicious server could send a crafted Socket.IO packet to the mobile client and execute arbitrary code on users' devices. Additionally, improper certificate validation (GHSA-72mh-269x-7mh5) enables MITM attacks on mobile Socket.IO connections.

## Suggested Fix

**Approach:** Replace with the official `socket.io-client` package.

**Affected files:** `apps/mobile/package.json`  
**Effort:** ~2 hours (package swap + import updates)  
**Risk:** Low — the official package exposes the same API. Connection logic and event handlers should not require changes.

```bash
cd apps/mobile
npm uninstall react-native-socket.io-client
npm install socket.io-client@4
```

Update all import statements:
```typescript
// Before
import io from 'react-native-socket.io-client';

// After
import { io } from 'socket.io-client';
```
