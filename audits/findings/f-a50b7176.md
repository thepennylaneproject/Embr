> **Status:** fixed_verified | **Severity:** major | **Priority:** P1 | **Type:** bug | **Confidence:** evidence

## Title

Tool endpoints use one shared workspace for all authenticated users

## Description

Tool endpoints now derive workspace paths from the authenticated JWT subject. WorkspacePolicy creates base_workspace_dir/user_id sandboxes, ToolService requires user_id on public file operations, and routes pass current_user.user_id from the auth dependency.

## Impact

Authenticated users no longer share one filesystem workspace through these tool endpoints; traversal checks also prevent paths from escaping the per-user sandbox.

## Suggested fix

Derive workspace_root from user_id under a base path via WorkspacePolicy.workspace_root_for(user_id)

**Affected files:** `backend/app/tools/policy.py`, `backend/app/tools/service.py`, `backend/app/api/routes/tools.py`, `backend/app/core/auth.py`
