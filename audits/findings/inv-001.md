# Finding: inv-001

> **Status:** open | **Severity:** Critical | **Priority:** P0 | **Type:** security | **Confidence:** Evidence

## Title

Next.js middleware authorization bypass — CVSS 9.1 (GHSA-f82v-jwr5-mffw)

## Description

The web app runs `next@14.2.5`, which contains a known authorization bypass in the Next.js middleware layer (GHSA-f82v-jwr5-mffw, CVSS 9.1). This vulnerability allows unauthenticated users to bypass middleware-based route protection and access routes that should require authentication. Because Embr's auth gating is implemented in Next.js middleware (`apps/web/src/middleware.ts`), all protected routes on the web frontend are potentially accessible without a valid session.

The fix is a version upgrade — no application code changes are required.

## Reproduction Steps

1. Identify a protected route gated by `apps/web/src/middleware.ts` (e.g., `/feed`, `/wallet`).
2. Using CVE exploitation techniques documented in GHSA-f82v-jwr5-mffw, craft a request that bypasses the middleware check.
3. Observe that the route responds with protected content rather than redirecting to `/login`.

_(Exact reproduction details are in the public CVE advisory.)_

## Impact

**Severity: CRITICAL**

Any authenticated route protected only by Next.js middleware can be accessed by unauthenticated users. For Embr this includes the entire logged-in product surface: feed, profile, wallet, gigs, and messaging.

## Suggested Fix

**Approach:** Version upgrade — no code changes required.

**Affected files:** `apps/web/package.json`  
**Effort:** < 1 hour  
**Risk:** Low — the upgrade is a patch version. Run `npm run build` and smoke-test auth flows after upgrading.

```bash
cd apps/web
npm install next@latest  # or pin to 14.2.25 minimum
npm run build
```
