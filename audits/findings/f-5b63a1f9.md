## Title

useComments mounted guard can lock updates off after cleanup

## Description

The hook sets isMountedRef.current = false in cleanup and never restores it on subsequent effect executions. If effect reruns (e.g., autoLoad toggle or remount behavior in same lifecycle), all guarded state updates are skipped.

## Proof Hooks

### [code_ref] Mounted flag set false in cleanup

- File: `apps/web/src/hooks/useComments.ts`

- Symbol: `useEffect cleanup`

- Lines: 268-271

### [code_ref] Guarded updates depend on mounted flag

- File: `apps/web/src/hooks/useComments.ts`

- Symbol: `loadComments/loadMore/update flows`

- Lines: 64-106


## Reproduction Steps

1. Use a component with useComments and trigger initial load.

2. Trigger a lifecycle path that runs cleanup then re-enters effect.

3. Call loadMore/createComment and observe server responses but no local state update.


## Impact

Comments and replies can silently stop updating.


## Suggested Fix

**Approach:** Set isMountedRef.current = true at effect start and false in cleanup, or replace with request cancellation/token approach.

**Affected files:** `apps/web/src/hooks/useComments.ts`

**Effort:** small

**Risk:** Low-medium; verify optimistic updates still revert correctly.
