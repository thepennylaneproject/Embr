> **Status:** open | **Severity:** Medium | **Priority:** P1 | **Type:** code-quality | **Confidence:** Evidence

## Title

9 console.log statements in production web source code

## Description

The web app contains 9 `console.log` calls in production source files that were left over from development. The API correctly uses NestJS's `Logger` service, but the web app did not follow the same discipline.

These log statements expose internal application state (WebSocket connection events, auth token refresh, message receipts) to any user who opens browser DevTools — a quality and minor security signal.

| File | Lines | Content |
|------|-------|---------|
| `apps/web/src/hooks/useMessaging.ts` | 99, 105, 119, 155, 177 | WebSocket connect, disconnect, message, typing events |
| `apps/web/src/contexts/AuthContext.tsx` | 78 | `'Token refreshed successfully'` |
| `apps/web/src/components/messaging/DMInbox.tsx` | 27, 32, 36 | New message, message read, typing indicator |
| `apps/web/src/pages/api/analytics/events.ts` | 39 | `[Analytics] Received N events` |

## Proof Hooks

## Reproduction Steps

1. Open the deployed web app in a browser.
2. Open DevTools → Console.
3. Log in and navigate to the messages section.
4. Observe WebSocket events being logged to the console.

## Impact

**Severity: MEDIUM**

Primarily a quality signal. Investors and technical reviewers who inspect the console will see implementation-level noise. The auth token refresh log (`Token refreshed successfully`) confirms token lifecycle to any observer.

## Suggested Fix

**Approach:** Remove all `console.log` calls from production web source. Use a dev-only logger utility for any diagnostics that are genuinely needed during development.

**Affected files:**
- `apps/web/src/hooks/useMessaging.ts`
- `apps/web/src/contexts/AuthContext.tsx`
- `apps/web/src/components/messaging/DMInbox.tsx`
- `apps/web/src/pages/api/analytics/events.ts`

**Effort:** 30 minutes  
**Risk:** None — removing diagnostic logs does not affect functionality.

**Optional improvement:** Create a `lib/logger.ts` utility:
```typescript
const logger = {
  debug: (...args: unknown[]) => {
    if (process.env.NODE_ENV === 'development') console.log(...args);
  }
};
```
Use `logger.debug(...)` in place of `console.log(...)` everywhere.
