## Title

Wallet creation swallows all persistence errors and falls back as if duplicate

## Description

createWallet catches every error and returns getWallet, assuming duplicate wallet race. Non-duplicate failures (DB outage, permission issues, malformed writes) are masked and misreported.

## Proof Hooks

### [code_ref] Broad catch with unconditional fallback

- File: `apps/api/src/core/monetization/services/wallet.service.ts`

- Symbol: `createWallet`

- Lines: 23-38


## Reproduction Steps

1. Force prisma.wallet.create to throw non-uniqueness error (e.g., transient DB failure).

2. Call createWallet.

3. Observe code attempts getWallet and returns misleading NotFound or unrelated error path.


## Impact

Root causes are hidden; operational incidents become harder to detect and recover.


## Suggested Fix

**Approach:** Catch and branch by known duplicate/constraint error code only; rethrow all other errors with context.

**Affected files:** `apps/api/src/core/monetization/services/wallet.service.ts`

**Effort:** small

**Risk:** Low.

- [ ] Unit test: non-duplicate DB error is rethrown.
