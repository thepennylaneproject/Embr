# Finding: f-dpl-0009

> **Status:** open | **Severity:** major | **Priority:** P1 | **Type:** enhancement | **Confidence:** evidence

## Title

Global error handling is incomplete for process-level and Next.js non-React failures

## Description

API only handles SIGTERM/SIGINT and lacks unhandledRejection/uncaughtException handlers. Web has a React error boundary in _app, but no explicit global Next.js error page path for non-boundary failures and no external reporting hook.

## Impact

Unhandled crashes and SSR/runtime failures can terminate process or degrade user sessions without reliable incident capture.


## Suggested Fix

**Approach:** Add process-level fatal error handlers and centralized reporting, plus explicit Next.js global error surfaces/reporting for non-component failures.

**Affected files:** `apps/api/src/main.ts` `apps/web/src/pages/app.tsx` `apps/web/src/components/ErrorBoundary.tsx` `apps/web/src/pages/error.tsx (new or equivalent)`

**Effort:** small

**Risk:** Fatal exception handlers must avoid masking crash loops; ensure graceful shutdown and supervised restart behavior.
