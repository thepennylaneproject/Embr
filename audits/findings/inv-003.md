> **Status:** open | **Severity:** Critical | **Priority:** P0 | **Type:** legal | **Confidence:** Evidence

## Title

No LICENSE file — contradictory license signals across README, package.json, and music-sdk

## Description

The repository has three mutually contradictory license signals and no `LICENSE` file to resolve the ambiguity:

1. Root `package.json` declares `"license": "MIT"`.
2. `packages/music-sdk/package.json` also declares `"license": "MIT"`.
3. `README.md` footer (line 47) states `"All rights reserved — © The Penny Lane Project"`.
4. No `LICENSE` file exists at the repository root.

Under most jurisdictions, code with no license file is "all rights reserved" by default even if `package.json` claims MIT. The contradictory signals mean the actual license terms are legally ambiguous. An investor's legal team performing IP due diligence will flag this as a blocker — they cannot advise investment into a project with unclear ownership and licensing.

## Impact

**Severity: CRITICAL for investor due diligence.**

Legal ambiguity around IP ownership is a standard deal-stopper in technical due diligence. Investors need clear answers to: "Who owns this code? Can I invest in it? Can it be used commercially?"

## Suggested Fix

**Approach:** Pick a license, create the file, and align all references. Two options:

**Option A — MIT (open source):**
1. Create `LICENSE` at repo root with standard MIT text. Substitute `[YEAR]` and `[OWNER]`.
2. Update `README.md` footer to: `MIT License — © 2026 [Owner Name / The Penny Lane Project]`.
3. Verify `package.json` and `packages/music-sdk/package.json` both say `"license": "MIT"`.

**Option B — Proprietary:**
1. Create `LICENSE` at repo root with "All rights reserved" text.
2. Update `package.json` to `"license": "UNLICENSED"`.
3. Update `packages/music-sdk/package.json` to `"license": "UNLICENSED"`.
4. Update `README.md` footer to match.

**Affected files:** `/LICENSE` (new), `README.md`, `package.json`, `packages/music-sdk/package.json`  
**Effort:** < 30 minutes  
**Risk:** None — this is documentation and metadata only.
