## Title

TypeScript strictness is disabled in shared and API build configs

## Description

Core TypeScript configs disable strict mode (strict: false) in both the monorepo base and API tsconfig. This allows latent runtime issues to pass build-time checks and reduces build confidence for production releases.

### [code_ref] API TS config disables strict mode and strictNullChecks

_(Optional for enhancements, debt, and questions.)_


## Impact

Builds can pass while null/typing defects remain, increasing production incident risk and making regressions harder to catch pre-deploy.


## Suggested Fix

**Approach:** Incrementally enable strict mode for API and shared TS configs, starting with strictNullChecks in high-risk modules and enforcing type fixes in CI.

**Affected files:** `tsconfig.base.json` `apps/api/tsconfig.json` `.github/workflows/ci.yml`

**Effort:** large

**Risk:** Turning strict mode on globally may initially fail many files; rollout should be staged to avoid delivery freeze.

- [ ] Run API build and smoke-test auth/content endpoints
