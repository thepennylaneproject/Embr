# Finding: inv-005

> **Status:** open | **Severity:** High | **Priority:** P1 | **Type:** security | **Confidence:** Evidence

## Title

Missing admin guards on two production endpoints (open TODOs in production code)

## Description

Two production endpoints in the API have `// TODO: Add admin guard` comments and are operating without the admin authorization guard applied. Any authenticated non-admin user can call these endpoints.

| File | Line | Endpoint |
|------|------|----------|
| `apps/api/src/core/notifications/notifications.controller.ts` | 133 | Admin notifications endpoint |
| `apps/api/src/core/monetization/controllers/stripe-connect.controller.ts` | 82 | Stripe Connect account deletion endpoint |

The Stripe Connect deletion endpoint is particularly risky — it can delete a connected account's Stripe relationship. Without an admin guard, any authenticated user could trigger this action against other accounts.

## Reproduction Steps

1. Obtain a valid JWT for a non-admin user (e.g., `user@embr.app`).
2. Call the admin notifications endpoint at line 133 — confirm it responds without 403.
3. Call the Stripe Connect deletion endpoint at line 82 — confirm it responds without 403.

## Impact

**Severity: HIGH**

Non-admin users can access admin notification management and trigger Stripe Connect account deletions. The Stripe endpoint represents a financial integrity risk — deletion of a connected account affects payout capability.

## Suggested Fix

**Approach:** Apply the existing admin guard (or ownership check) to both endpoints.

**Affected files:**
- `apps/api/src/core/notifications/notifications.controller.ts`
- `apps/api/src/core/monetization/controllers/stripe-connect.controller.ts`

**Effort:** 30–60 minutes  
**Risk:** Low — existing guard infrastructure is already in place. This is applying it, not building it.

```typescript
// Locate the admin roles guard (likely one of these patterns):
@Roles(UserRole.ADMIN)
@UseGuards(JwtAuthGuard, RolesGuard)

// For the Stripe endpoint (which may serve both admin and self-deletion):
// Check: req.user.id === targetUserId || req.user.role === UserRole.ADMIN
```

Remove both TODO comments after applying the guards.

## Tests Needed

- [ ] Non-admin user receives 403 (or 401) on Stripe Connect deletion endpoint
- [ ] Admin user can still access both endpoints
- [ ] For Stripe endpoint: if self-deletion is intended, a user can delete their own account but not others'
