# Finding: f-dpl-0007

> **Status:** open | **Severity:** major | **Priority:** P1 | **Type:** enhancement | **Confidence:** evidence

## Title

Required environment variables are not validated at startup

## Description

Configuration is loaded from env files but no startup schema validation is configured. Runtime-critical values are read directly from process.env, so misconfiguration fails late and inconsistently.

## Proof Hooks

### [code_ref] ConfigModule.forRoot has no validationSchema/validate hook

- Symbol: `ConfigModule.forRoot`

- Lines: 28-31

### [code_ref] Bootstrap directly consumes process.env values

- File: `apps/api/src/main.ts`

- Symbol: `bootstrap`

- Lines: 41-52

## Impact

Bad or missing env values surface as runtime incidents instead of fast-fail startup errors, increasing MTTR and deployment uncertainty.


## Suggested Fix

**Approach:** Add centralized env validation (Joi/Zod) in ConfigModule startup and fail boot on missing/invalid required variables.

**Affected files:** `apps/api/src/app.module.ts` `apps/api/src/config/env.validation.ts (new)` `.env.example` `apps/api/.env.example` `apps/web/.env.example`

**Effort:** small

**Risk:** Initial enforcement may break existing local setups until env files are normalized.
